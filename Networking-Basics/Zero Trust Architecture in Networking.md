## What is Zero Trust Architecture?  
- **Definition:** A security model that assumes no user or device is trusted by default, even inside the network.  
- **Purpose:** Continuously verifies every access request → reduces risk of internal and external threats.  
- **Analogy:** Like checking ID at every door instead of trusting someone just because they entered the building once.  

---

## The 4 Core Steps of Zero Trust Operation  

### 1. Identity Verification (Step 1)  
- **Function:** User/device identity is verified before access.  
- **Role:** Ensures only authenticated entities proceed.  
- **Examples:** MFA, biometrics, certificates.  

---

### 2. Device Validation (Step 2)  
- **Function:** Checks device health and compliance.  
- **Role:** Prevents compromised devices from accessing resources.  
- **Examples:** Antivirus status, OS updates.  

---

### 3. Access Control (Step 3)  
- **Function:** Grants least-privilege access based on policies.  
- **Role:** Limits user permissions to required resources only.  
- **Examples:** Role-based access control (RBAC).  

---

### 4. Continuous Monitoring (Step 4)  
- **Function:** Tracks activity and re-validates trust continuously.  
- **Role:** Detects anomalies and revokes access if needed.  
- **Examples:** Behavior analytics, session monitoring.  

---

## Key Principles  
- **Never Trust, Always Verify**  
- **Least Privilege Access**  
- **Assume Breach Mindset**  

---

## Why It Matters  
- **Security:** Reduces attack surface.  
- **Data Protection:** Prevents unauthorized access.  
- **Modern Networks:** Essential for cloud and remote work environments.  

---

## Quick Recap (Mnemonic)  
- **Verify → Validate → Control → Monitor**  
  - **Identity → Device → Access → Activity**
