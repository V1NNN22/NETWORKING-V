## What is a Routing Table?  
- **Definition:** A data structure stored in routers and operating systems that determines where network packets should be forwarded.  
- **Purpose:** Helps routers decide the best path for packets to reach their destination network.  
- **Analogy:** Like a navigation map that tells a driver which road to take to reach a destination.  

---

## The 4 Core Components of a Routing Table  

### 1. Destination Network  
- **Function:** Specifies the target network or host.  
- **Role:** Identifies where the packet needs to go.  
- **Examples:** `192.168.1.0/24`.  

---

### 2. Next Hop  
- **Function:** Indicates the next router or gateway for forwarding packets.  
- **Role:** Defines the immediate device that should receive the packet.  
- **Examples:** Default gateway like `192.168.1.1`.  

---

### 3. Interface  
- **Function:** Specifies which network interface should be used to send the packet.  
- **Role:** Determines the physical or virtual path used.  
- **Examples:** `eth0`, `wlan0`.  

---

### 4. Metric (Cost)  
- **Function:** Represents the cost of a route.  
- **Role:** Used to choose the best route when multiple paths exist.  
- **Examples:** Lower metric value preferred.  

---

## Example Routing Table Entry  

| Destination | Gateway | Interface | Metric |
|-------------|---------|----------|--------|
| 0.0.0.0/0 | 192.168.1.1 | eth0 | 10 |
| 192.168.1.0/24 | 0.0.0.0 | eth0 | 0 |

---

## Why It Matters  
- **Packet Forwarding:** Enables routers to move data across networks.  
- **Network Efficiency:** Ensures shortest or best path is selected.  
- **Troubleshooting:** Helps diagnose routing problems.  

---

## Quick Recap (Mnemonic)  
- **Destination → Gateway → Interface → Cost**  
  - **Where → Through → Which Port → Best Path**
