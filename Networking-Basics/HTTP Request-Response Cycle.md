## What is HTTP Request-Response Cycle?  
- **Definition:** A communication process where a client sends a request to a server and the server responds with the requested resource.  
- **Purpose:** Enables web communication between browsers and servers → foundation of how websites work.  
- **Analogy:** Like ordering food in a restaurant where you request (order) and the kitchen responds (serves).  

---

## The 4 Core Steps of HTTP Cycle  

### 1. Client Request (Step 1)  
- **Function:** Browser sends an HTTP request to the server.  
- **Role:** Includes method, headers, and optional body.  
- **Examples:** GET /index.html HTTP/1.1  

---

### 2. DNS Resolution (Step 2)  
- **Function:** Domain name is resolved to an IP address.  
- **Role:** Allows client to locate server.  
- **Examples:** example.com → 93.184.216.34  

---

### 3. Server Processing (Step 3)  
- **Function:** Server processes the request.  
- **Role:** Fetches resource or executes backend logic.  
- **Examples:** Web server retrieving HTML page.  

---

### 4. Server Response (Step 4)  
- **Function:** Server sends HTTP response back to client.  
- **Role:** Includes status code, headers, and content.  
- **Examples:** 200 OK with webpage data.  

---

## Key HTTP Methods  
- **GET:** Retrieve data  
- **POST:** Send data  
- **PUT:** Update data  
- **DELETE:** Remove data  

---

## Why It Matters  
- **Web Functionality:** Core of all web applications.  
- **Client-Server Model:** Enables communication over internet.  
- **Debugging:** Helps understand API and web issues.  

---

## Quick Recap (Mnemonic)  
- **Request → Resolve → Process → Respond**  
  - **Client → DNS → Server → Response**
