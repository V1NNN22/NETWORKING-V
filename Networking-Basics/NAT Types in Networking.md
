## What are NAT Types?  
- **Definition:** Different behaviors of Network Address Translation that define how internal devices communicate with external networks.  
- **Purpose:** Controls connectivity, accessibility, and security level of devices behind NAT.  
- **Analogy:** Like different levels of access control in a building deciding who can enter or be contacted from outside.  

---

## The 4 Types of NAT  

### 1. Full Cone NAT (Step 1)  
- **Function:** Once mapped, any external host can send packets to internal device.  
- **Role:** Least restrictive NAT type.  
- **Examples:** Open connectivity for gaming/VoIP.  

---

### 2. Restricted Cone NAT (Step 2)  
- **Function:** External hosts can send packets only if internal device previously sent request to them.  
- **Role:** Moderate restriction.  
- **Examples:** Basic firewall-like behavior.  

---

### 3. Port Restricted Cone NAT (Step 3)  
- **Function:** Similar to restricted NAT but also checks port numbers.  
- **Role:** More strict filtering.  
- **Examples:** Only specific IP + port allowed.  

---

### 4. Symmetric NAT (Step 4)  
- **Function:** Each outgoing request creates unique mapping.  
- **Role:** Most restrictive NAT type.  
- **Examples:** Difficult for P2P communication.  

---

## Key Differences  

| NAT Type | Restriction Level | Incoming Access |
|----------|------------------|-----------------|
| Full Cone | Low | Any host |
| Restricted | Medium | Known IPs only |
| Port Restricted | High | Known IP + Port |
| Symmetric | Very High | Unique per session |

---

## Why It Matters  
- **Connectivity:** Affects gaming, VoIP, P2P apps.  
- **Security:** More restriction = more protection.  
- **Troubleshooting:** Helps diagnose connection issues.  

---

## Quick Recap (Mnemonic)  
- **Open → Limited → Strict → Strictest**  
  - **Full → Restricted → Port → Symmetric**
