## What is Flow Control?  
- **Definition:** A mechanism that ensures a sender does not overwhelm a receiver by sending data too fast.  
- **Purpose:** Maintains balance between sender and receiver → prevents data loss and buffer overflow.  
- **Analogy:** Like controlling water flow in a pipe so the container doesn’t overflow.  

---

## The 4 Core Steps of Flow Control  

### 1. Data Transmission (Step 1)  
- **Function:** Sender starts transmitting data packets.  
- **Role:** Sends data based on allowed capacity.  
- **Examples:** TCP sending segments to receiver.  

---

### 2. Receiver Capacity Check (Step 2)  
- **Function:** Receiver evaluates its buffer space.  
- **Role:** Determines how much data it can handle.  
- **Examples:** Available buffer size in TCP window.  

---

### 3. Feedback to Sender (Step 3)  
- **Function:** Receiver sends window size information.  
- **Role:** Informs sender about how much data can be sent.  
- **Examples:** TCP Window Size field.  

---

### 4. Transmission Adjustment (Step 4)  
- **Function:** Sender adjusts sending rate based on feedback.  
- **Role:** Prevents overwhelming receiver.  
- **Examples:** Sliding window protocol.  

---

## Flow Control Techniques  
- **Stop-and-Wait:** Send one packet, wait for ACK.  
- **Sliding Window:** Send multiple packets before waiting.  

---

## Why It Matters  
- **Reliability:** Prevents data loss.  
- **Efficiency:** Optimizes data transfer rate.  
- **Stability:** Maintains smooth communication.  

---

## Quick Recap (Mnemonic)  
- **Send → Check → Inform → Adjust**  
  - **Data → Buffer → Window → Control**
